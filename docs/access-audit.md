be connected to each teamspace/area you want it to reach; do **not** assume it
bypasses teamspace integration blocks.

## How integration access works in Notion

A token can only see a page or database if **both** are true:

1. The **teamspace** that contains it allows integrations (teamspace setting,
   controlled by teamspace owners).
2. The integration has been added as a **connection** to that page/database
   (`•••` → Connections → add the integration), or to a parent page and
   cascaded to children.

If a teamspace has integrations disabled, **nothing inside it** can be reached
by an integration/PAT — including all sub-pages and databases.

## Confirming Ecosystem access for the PAT

To test whether the PAT can reach the Ecosystem teamspace, give it a specific
Ecosystem page or database URL and probe:

```bash
set -a; source .env; set +a
ntn pages get <ecosystem-page-id>          # returns content if reachable, 404 if not
ntn datasources resolve <ecosystem-db-id>  # same idea for a database
```

If it 404s, the teamspace block applies to the PAT too — Ecosystem work must
stay on the Notion MCP tools (which run under your personal OAuth user with
full access), not the PAT/CLI.
